# 优化策略方案

## 问题诊断

**根因**：CMA-ES 依赖目标函数的局部平滑性。但 `hx_unmatched` 目标在分位数变化导致子循环拓扑改变时产生跳变——协方差矩阵失效，大量代数零改进。

```
分位数微调 → 子循环枚举变化 → 记录数突变 → HX 匹配跳变 → obj 跳变
```

**症状**：所有测试中 r0 收敛（~30 代），r1-r4 完全无效（0 改进/50 代）。

---

## 方案 A：分层搜索（推荐）

### 思路

```
外层: LHS 采样分位组合 → 冻结拓扑
内层: CMA 固定拓扑优化流量 → 快速收敛
最终: top-K 用 CoolProp 验证
```

### 参数

| 层 | 方法 | 参数 |
|------|------|------|
| 外层 | LHS 采样 | 200-500 点覆盖分位空间 |
| 内层 | CMA | maxiter=20, restarts=2, interp He |

### 优缺点

| 优点 | 代价 |
|------|------|
| 拓扑空间被**明确定界探索**，不依赖梯度 | 代码改动 ~200 行 |
| 内层 obj 连续 → CMA 快速稳定，彻底消除零改进代数 | 总 eval 数 = 外层采样 × 内层代数 × popsize |
| interp He 在内层大量使用，最终 CoolProp 验证 | 暴力枚举对 3T+3P 不适用，需降维 |
| 解耦清晰，便于分析"哪个拓扑好" | — |

### 估算（2T+2P）

外层 LHS 300 × 内层 CMA 20gen×17pop×2restarts×10ms / 6workers ≈ **57s**

---

## 方案 B：贝叶斯优化

### 思路

用 GP 代理模型（optuna TPE / scikit-optimize GP-EI）替代 CMA-ES，对不连续/噪声目标鲁棒。

### 参数

| 项 | 选择 |
|------|------|
| 库 | optuna（TPE 采样器）或 scikit-optimize |
| 采样器 | TPE（维度 ≤15）或 GP（维度 ≤10） |
| n_trials | 500-1000 |

### 优缺点

| 优点 | 代价 |
|------|------|
| 原生处理不连续、多局部最优目标 | dim>20 时 GP 代价 O(n³)，TPE 退化 |
| 改动量 ~100 行 | 引入新依赖，超参数多 |
| 采样效率高 | 对纯连续变量不如 CMA 精确 |

---

## 方案 C：换目标——系统净功

### 思路

`obj = -net_mechanical`，HX 匹配作为内部硬约束（dT_min），非优化目标。

### 优缺点

| 优点 | 代价 |
|------|------|
| 目标物理意义清晰 | HX 匹配不再被直接优化 |
| 代码量 ~10 行 | 冷热源功率固定，净功优化梯度可能不显著 |
| 可与 A、E 组合做多目标 | skip_pinch 逻辑需调整 |

---

## 方案 D：多目标（NSGA-II）

### 思路

同时优化 `hx_unmatched` + `net_mech`，生成 Pareto 前沿。

### 参数

| 项 | 选择 |
|------|------|
| 库 | pymoo 或 DEAP |
| 种群 | 100 × 100 代 = 10k eval |

### 优缺点

| 优点 | 代价 |
|------|------|
| 不做武断取舍，交付 Pareto 前沿 | 代码量 ~300 行 |
| 交互式决策 | 10k eval，引入新依赖 |
| — | 结果解读复杂 |

---

## 方案 E：工程约束+接受天花板

### 思路

obj=0.044 已足够。优化转向工程可行指标：

| 约束 | 值 | 原因 |
|------|:--|------|
| `max_group_size=2` | 换热器内最多 2 冷边 | 降低机械复杂度 |
| `t_min >= 280K` | 避免深冷 | 无需低温制冷设备 |
| `p_max <= 8000 kPa` | 降低管路压力等级 | 减少设备成本 |
| N+1 惩罚 | 已有 | 减少碎片化未匹配 |

### 优缺点

| 优点 | 代价 |
|------|------|
| 改动量 ~30 行 | 本质换天花板 |
| 结果可施工 | 需要领域知识定约束 |
| 与方案 A 可组合 | — |

## 推荐方案

**A（分层搜索）+ E（工程约束）**：

1. 外层 LHS 采样分位空间
2. 内层 CMA（interp 快跑，top-10 用 CoolProp 验证）
3. 目标含 N+1 惩罚 + t_min≥280K 约束
4. 最终输出 Pareto："匹配率 vs 工程可行性"
